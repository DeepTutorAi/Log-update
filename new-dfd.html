New DFD blueprint

Wave story and evolution path

เหมาะสำหรับอ่านว่าแต่ละ wave เปลี่ยนอะไร เพิ่มอะไร และระบบโตขึ้นอย่างไรโดยไม่ทำลายของเดิม

  • โฟกัสที่การเปลี่ยนแปลงตาม release wave
  • อ่านจาก baseline → learning → crew → evolution
  • ตอบคำถามแบบ “wave นี้เพิ่มอะไรกับทำไมต้องมี”

Wave 1 · MVP Closure

ปิดช่องว่างพื้นฐานให้ระบบนิ่งก่อนขยับไปสู่ self-learning

Wave 1 ทำให้ baseline EA แข็งแรงขึ้นโดยไม่แตะพฤติกรรมเดิม: เพิ่ม module ที่ README เคลมไว้แต่ยังไม่มีจริง และตั้งทุก feature ใหม่เป็น opt-in

Position Governor Risk Hardening Symbol Meta Cache Regression Tests

What changed

  • เพิ่ม `PositionGovernor` สำหรับ trailing, partial TP, breakeven และ net-exit basket
  • เพิ่ม risk hardening เช่น streak guard, gap guard และ equity HWM
  • เพิ่ม `SymbolMetaCache` เพื่อลดการเรียก broker metadata ซ้ำ ๆ
  • เพิ่ม regression suite เพื่อกัน behavior drift

Why it matters

  • baseline ยังทำงานเหมือนเดิมถ้าไม่เปิด flag
  • position lifecycle แยกจาก entry logic ชัดเจน
  • ก่อน wave 2 ระบบมี safety floor ที่น่าเชื่อถือกว่าเดิม
Visible effect: เพิ่ม control และความปลอดภัย แต่ไม่เปลี่ยน default live behavior
Wave 2 · Online Learning

สร้างวงจรเรียนรู้ที่ปลอดภัยด้วย registry, shadow mode และ approval gate

Wave 2 เปลี่ยนจาก config แบบ compile-time ไปสู่ระบบที่ปรับ runtime ได้ แต่ยังต้องผ่าน shadow test และ human approval ก่อน commit จริง

Param Registry Shadow Mode Approve UI Rollback

What changed

  • สร้าง `ParamRegistry` ให้ EA อ่าน override ได้ใน `OnInit()`
  • เพิ่ม shadow mode เพื่อทดสอบ param ใหม่แบบไม่แตะ live order
  • สร้าง reflection → recommendation pipeline สำหรับเสนอค่าปรับปรุง
  • เพิ่ม dashboard approve UI และ safety gates

Why it matters

  • ระบบเริ่ม evolve ได้โดยยังมีคนคุมประตู
  • แต่ละ proposal มี audit trail และย้อนกลับได้
  • ความต่างจากหน้าเดิมคือเน้น learning loop ไม่ใช่ runtime flow
Visible effect: เริ่มมี “แนะนำ → ทดสอบ → อนุมัติ” แทนการแก้ค่าด้วยมืออย่างเดียว
Wave 3 · Crew Router

ยกระดับ AI ให้เลือก skill ตาม context และมี planner/critic consensus

Wave 3 เปลี่ยน AI layer จาก webhook ครั้งเดียวเป็น crew ที่รู้จัก skill ของตัวเอง ทำให้ decision routing ฉลาดขึ้น และลดค่าเรียกซ้ำด้วย cache

Skill Registry Planner / Critic Cache Layer Trust Scoring

What changed

  • เพิ่ม skill registry สำหรับ regime-specific brains
  • เพิ่ม planner + critic crew เพื่อหาคำตอบที่มี consensus มากขึ้น
  • เพิ่ม cache layer เพื่อลด webhook calls ซ้ำ ๆ
  • เพิ่ม per-skill trust score เพื่อปรับ routing จาก outcome ย้อนกลับ

Why it matters

  • AI ไม่ใช่ก้อนเดียวอีกต่อไป แต่เป็นระบบ skill-aware
  • ลดโอกาสตอบมั่วจากการใช้ context ผิด regime
  • หน้าใหม่ควรเล่า evolution นี้ให้เห็นว่าระบบ “คิด” ต่างจาก wave 2 อย่างไร
Visible effect: AI layer แบ่งหน้าที่ชัดขึ้น และเริ่มมีการชั่งน้ำหนักระหว่าง skill
Wave 4 · Evolution Engine

เพิ่มระบบเสนอค่าปรับปรุงอัตโนมัติผ่าน GA, WFO และ Monte Carlo

Wave 4 คือชั้น evolution ที่ค้นหา parameter set ใหม่จากข้อมูลย้อนหลัง แต่ยังไม่ auto-commit — ทุกข้อเสนอจะผ่าน validation หนัก ๆ และส่งให้คนอนุมัติเท่านั้น

GA Runner WFO Gate Monte Carlo Dashboard Queue

What changed

  • สร้าง GA runner สำหรับค้นหา candidate param sets
  • ผ่าน WFO gate และ Monte Carlo validation ก่อนเสนอผล
  • scheduler + notification ช่วยรันเป็นรอบตามเวลา
  • ผลลัพธ์ไปอยู่ใน dashboard queue เดียวกับ approval flow ของ wave 2

Why it matters

  • evolution กลายเป็น proposal layer ไม่ใช่ auto-trading layer
  • human approval ยังคงเป็นจุดสุดท้ายก่อน commit
  • หน้านี้จึงต่างจาก legacy DFD เพราะเล่าอนาคตของระบบ ไม่ใช่ runtime map
Visible effect: ระบบเสนอ candidate ที่ผ่าน validation ได้ แต่ยังเคารพ human approval
